# 北京师范大学招生宣传智能体系统开发需求分析报告（修订版）

## 一、整体目标  
构建一个面向本科及研究生招生的智能对话系统（智能体），以多样化、智能化、富有人文关怀的方式服务考生与家长，提升北京师范大学招生宣传的专业性、亲和力与影响力。系统需基于权威知识库，实现精准问答、情绪支持、内容引导，并兼顾安全性、可维护性与用户体验。

---

## 二、核心功能需求

### 1. 用户注册与登录
- 用户需注册后方可使用系统，注册登录流程须简便快捷。
- **必须使用中国手机号 + 短信验证码登录**，确保身份真实性与操作便捷性。
- 管理员账号只能通过超级管理员手动添加才能登录，需要更强的身份验证保证系统的安全性。

### 2. 智能对话主体功能
- 支持自然语言交互，覆盖招生政策、专业介绍、报考流程、校园生活等高频场景。
- 针对不同用户角色（如高考生、考研生、国际学生、家长）提供差异化回答策略。
- 在无法准确回答或涉及高风险问题时，主动引导用户联系招生办老师，并提供联系方式。

### 2. AI幻觉防控机制
- **风险分级机制**：对用户提问进行风险评估（低/中/高），高风险问题限制自由生成，仅返回审核通过的标准答案或引导语。
- **双模型审查**：主模型生成回答后，由轻量级审查模型校验事实一致性与合规性。
- **敏感词过滤**：内置敏感词库，自动拦截或转人工处理不当内容。支持有多个敏感词库，支持管理员快捷查看、编辑敏感词库，以及敏感词库的内容。
- **引用溯源**：关键信息附带知识库来源标识（如“根据2025年招生简章…”）。

### 3. 模糊意图识别与追问引导
- 当用户提问模糊或信息不足时，系统基于历史对话与统计关联度，智能推荐2–3个可能的细化问题供用户选择（如：“您是想了解心理学专业的课程设置，还是就业方向？”）。
- 支持上下文记忆，在多轮对话中保持语义连贯。

### 4. 知识库与内容管理
- 建立结构化招生知识库，支持文本、政策文件、FAQ等格式。
- 语料更新流程：管理员提交 → 部门负责人审核 → 自动切片与Embedding → 生效上线。
- 所有知识条目标注更新时间、责任人、生效状态，确保可追溯。
- 知识库支持多种格式的文件上传，PDF，Word，txt，markdown等。

### 5. 多媒体内容集成
- 对接学校官方公众号/媒体库，展示经审核的校园图片、宣传片、VR校园导览等。提供多媒体来源管理界面。
- 构建多媒体资源库，支持按关键词或语义相似度检索并嵌入对话（如用户问“宿舍什么样？”，返回实景照片+视频链接）。
- **严禁AI生成图像或视频**，所有视觉内容必须来自官方素材。

### 6. 时间感知与话术动态调整
- 系统内置招生日历，根据当前月份自动切换话术风格与重点内容：
  - **1–6月（备考期）**：侧重激励、备考建议、专业前景；
  - **6–7月（高考后/报名期）**：聚焦志愿填报、分数线预测、报名指南；
  - **8–9月（录取查询期）**：提供录取结果查询指引、入学准备清单；
  - **全年常态**：校园文化、师资力量、国际交流等通用内容。

### 7. 人文关怀与情绪支持
- 集成基础情感识别模块，当检测到用户表达焦虑、迷茫、挫败等情绪时，触发安慰性话术（如“考试压力大是很正常的，北师大欢迎每一位努力的你！”）。
- 提供心理支持资源链接（如校心理咨询中心、减压指南等）。

### 8. 多模态输入支持
- 允许用户上传图片（如成绩单截图）、PDF、DOC等文件。
- 后端解析文档内容，提取关键信息（如分数、专业名称），结合知识库生成针对性回复。
- 文件存储加密，保留元数据（上传时间、用户ID、文件类型、解析结果）。

### 9. 系统管理与运维
- **多模型接入**：支持配置多个大模型（如通义千问、GLM、本地部署模型），管理员可切换主用模型或启用负载均衡。
- **详细日志审计**：
  - 记录每条对话：用户ID、IP、时间戳、提问内容、模型响应、所用模型版本、知识库命中记录；
  - 记录所有文件上传：文件名、哈希值、解析状态、关联对话ID；
  - 日志支持导出与可视化分析。
- **权限体系**：
  - 角色分级：超级管理员、内容审核员、普通管理员、招生老师；
  - 细粒度权限控制（如仅审核员可批准语料更新）。

## 三、前端体验与设计需求

### 1. 美观性与品牌一致性
- 整体UI/UX设计深度融合**北京师范大学视觉识别系统（VIS）**：
  - 主色调采用北师大标准色（学术蓝）；
  - 融入“木铎”“京师学堂”“主楼剪影”等文化元素作为装饰图形；
  - 字体优先使用思源黑体或学校官方字体，体现庄重与亲和并存的气质。
- 动效流畅、界面简洁，避免信息过载，突出核心功能。

### 2. 易用性与无障碍设计
- 首页清晰展示“十大高频问题”卡片，支持一键跳转问答。
- 对话界面支持语音输入、快捷按钮（如“我想了解心理学”“如何查分？”）。
- 提供夜间模式、字体大小调节，符合WCAG 2.1无障碍标准。

### 3. 多端自适应支持
- 采用响应式设计，完美适配：
  - **PC端**：宽屏布局，支持多窗口操作、文件拖拽上传；
  - **移动端（iOS/Android）**：触控优化，对话气泡适配小屏，支持横竖屏切换；
  - **平板端**：介于两者之间，兼顾阅读与交互效率。
- 前端技术栈：Vue 3 + Vite + TypeScript + Pinia + Element Plus / Naive UI（按设计风格选型），确保高性能与可维护性。
- 前端涉及参考当前的主流大语言模型对话界面，用户需要能清晰查看对话历史，编辑对话历史的标题（对话历史的标题需要根据对话内容自动生成并更新）。

### 4. 加载性能与稳定性
- 首屏加载时间 ≤1.5 秒；
- 对话响应延迟 ≤2 秒（含后端处理）；
- 支持离线缓存部分静态资源（如高频问题、校园图片缩略图）。

## 四、非功能性需求

| 类别         | 要求说明 |
|--------------|--------|
| 安全性       | 用户数据加密传输（HTTPS）、敏感操作二次验证、防刷防爬机制 |
| 可维护性     | 模块化代码结构，完善的API文档与运维监控面板 |
| 合规性       | 符合《个人信息保护法》《数据安全法》，不存储用户身份敏感信息 |
